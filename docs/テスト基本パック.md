# エモリー 動作テスト「基本パック」

**何か対応したら必ずこのパックを通すこと。** 見た目の確認だけ・型チェックだけでは
過去に何度もデグレを見逃した。実画面をヘッドレスで実際に操作して、主要動作・例外
事象・性能まで自動アサートする。

## 実行方法

```bash
npm run check     # ① 静的: 型(tsc) ＋ 仕様(重なり/隙間/非格子)
npm run e2e       # ② 実機: ヘッドレスChromeで主要インタラクションを自動検証
# ソース変更後にビルドからやり直す場合:
npm run e2e:build
```

`npm run e2e` は `e2e/run.sh` 経由で
1. （必要なら）`e2e/` に puppeteer を用意（Chrome は storage.googleapis.com から取得）
2. `expo export` で web ビルド
3. ローカル配信（`http://localhost:8099/Emory/`）
4. `e2e/pack.js` を実行（1つでも失敗で exit 1）

計測には本番にも入っている軽量フックを使用（実害なし）:
`window.__emoryThrows`（投擲数）/ `window.__emoryGoals`（バスケ得点）/
`window.__emorySurf(x)`（指x位置の局所表面の画面y）/
`window.__emoryLastDrop`（直近に落としたボールの沈み込み sink 等＝貫通検出）。

## チェック項目（基本メニュー）

| # | 項目 | 期待 |
|---|------|------|
| T1 | 上の空間タップ | 1タップ=1生成 |
| T2 | 連続タップ(45ms) | 8回→8生成（取りこぼし無し） |
| T3 | 絵文字層タップ | 生成0（層内に湧かない・すり抜け無し） |
| T4 | フリック | その方向へ投擲（+1） |
| T5 | 層を掴んでドラッグ | スクロールのみ・生成0 |
| T6 | 演出切替ボタン | 存在・機能 |
| T7 | すり抜け検査 | 画面下端を大きく超える残留ボール0 |
| T8 | 性能(4xスロットル,30投) | 中央値 ≥ 40fps |
| T9 | バスケ描画 | ゴールが表示される |
| T10 | バスケ中央通過 | 上から中央を抜けると得点 |
| T11 | バスケ リム命中 | 縁ではバウンドし得点しない |
| T12 | エラー監視 | console/page エラー0 |
| T13 | 貫通検査(高所) | 色々な場所/高さから落としても層を貫通して下まで落ちない |
| T14 | 貫通検査(連投) | 中心へ高速連投しても掘り抜けず沈み込みが抑えられる(__emoryMaxBallSink) |
| T15 | 重なり検査 | シードの山にボール同士の明確な重なりが無い(中心間距離 >= 見た目の径×0.75) |
| T16 | すり抜け(凸凹) | 偏った/凸凹な表面の各所へ落としても他球の真下に潜り込まない |
| T17 | 永続キャッシュ | 初回ロードで baked レイアウトを保存(miss)、再読込で命中(hit)し物理 settle を回さず同一レイアウトを復元する(Phase 1) |
| T18 | 履歴保持 | 旧上限180を超えて積んでも固定データを破棄しない(=下層/記録が消えない, #13/#11)。窓スライス描画で fps も落ちない(Phase 2) |

## 運用ルール

- **対応 → `npm run check` → `npm run e2e` が全て緑 → コミット/デプロイ。**
- 失敗したら原因を実測で特定して修正、再度パックを通してからデプロイ。
- 新しい例外事象を見つけたら、再発防止のため本パックにチェックを追加する。
- 数値で測れない見た目は、`e2e/pack.js` にスクショ保存を足して目視併用。

## 注意（テスト時の落とし穴）

- 画面座標で「層の中」を狙うときは、**静止後に実ボール中心**を狙う。カメラ追従中に
  撮った座標は古くなり、誤って空間タップ扱いになる（=偽の生成）。
- タップは `page.touchscreen.tap` を使う（`mouse.click` は連打でダブルクリックに合体する）。
- ピッカー（上部の感情選択）は画面上端〜約172pxを覆う。投擲はその下の y を狙う。
- 山の表面は凸凹。投擲ゾーン/層の境界は**列ごとの局所表面**（`__emorySurf(x)`）で判定する。
