# エモリー（Emory）システム設計書

最終更新: 2026-06-13 / 対象: MVP (v0.1.0)

---

## 1. 概要

**エモリー** は、日々の感情を **3Dの丸い感情ボール** として記録し、スマホという箱の中に
**物理的に溜めていく** 感情ログアプリ。

日記のように文章で内省するのではなく、

1. 今の気分に近い感情ボールを選ぶ
2. 画面上で上方向にドラッグ／フリックして投げ込む
3. ボールが画面下部に落ちて積み重なる
4. 下にスクロールすると、過去の感情がギチギチに詰まって見える

という **「触れる・溜まる・眺める」** 体験を中心に据える。感情を数値化・分析する
ツールではなく、感情ログを情緒的なオブジェクトとして蓄積するアプリ。

### コアコンセプト
> 「感情を、スマホの中に物理的に溜めていく」

## 2. デザイン方針

- **トーン**: Apple標準デザイン風 × 北欧 × アースカラー
- **背景**: ベージュ／アイボリー／グレージュ系の温かいオフホワイト（`#F3ECE0` 基調）
- **感情ボール**: マットな陶器/フェルト質感の3D球体。シンプルな点目＋線の口。可愛すぎず大人っぽい
- **アーカイブ**: カードやグリッドではなく、画面いっぱいにボールが密に詰まった見た目。右側に小さなピル型の日付ラベル＋薄い点線境界
- **避けるもの**: キラキラ演出、子ども向けゲーム感、ガラス容器、過剰な説明文、ビビッドな色、整列しすぎたグリッド

## 3. 技術スタック

| 領域 | 採用 | 備考 |
|------|------|------|
| フレームワーク | **Expo (managed) + React Native + TypeScript** | iOS / Android / Web 対応 |
| 物理演算 | **matter-js** | 純JS物理エンジン。投擲＋下部プールに適用 |
| ジェスチャ | **react-native-gesture-handler** | スリングショット式のドラッグ/フリック検出 |
| アニメーション | react-native-reanimated | （将来の演出拡張用に同梱） |
| 描画 | **react-native-svg** | ターゲット・軌道線・日付点線・ヘッダーアイコン |
| 永続化 | **@react-native-async-storage/async-storage** | 端末ローカル保存 |
| 画像 | 実素材PNG（透過） | 8感情 × 4バリエーション = 32枚 |

> **設計判断**: ライブ物理は「追加画面」の能動的な投擲＋直近の溜まりだけに限定し、
> アーカイブは静的な密パックで描画する。アーカイブ要件は「前面から密に詰まった印象」で
> あり奥行き物理ではないため、パフォーマンスと体験を両立できる。

## 5. データモデル

AsyncStorage に JSON で保存。

| キー | 型 | 内容 |
|------|----|----|
| `emory.entries` | `EmotionEntry[]` | 感情エントリの配列 |
| `emory.throws` | `number[]` | 投擲タイムスタンプ（制限管理用） |

```ts
interface EmotionEntry {
  id: string;          // 一意ID
  emotion: EmotionKey; // 8感情のいずれか
  variation: number;   // 使用したボール画像 (0..3)
  color: string;       // 代表色（UIアクセント用）
  memo?: string;       // 一言メモ（任意）
  createdAt: string;   // ISO日時
}
```

### 感情カテゴリ（8種）
| key | ラベル | グループ | 代表色 |
|-----|--------|----------|--------|
| happy | うれしい | positive | `#D78D27` |
| excited | わくわく | positive | `#C86939` |
| calm | 落ち着く | neutral | `#E2D7C9` |
| relaxed | リラックス | neutral | `#DFD3C8` |
| tired | つかれた | negative | `#838E9B` |
| sad | 悲しい | negative | `#97A2AA` |
| anxious | 不安 | negative | `#909CAA` |
| irritated | イライラ | negative | `#9E9AA5` |

各感情は4バリエーションのボール素材を持ち、エントリ生成時に id をシードに
決定論的に1枚を選ぶ（`variationForId`）。

## 6. 画面仕様

### 6.1 追加画面（AddScreen）
- **ヘッダー**: 左=顔アイコン、中央=「エモリー」、右=カレンダー/メニュー。残り投擲数を控えめなドット列で表示
- **ターゲット**: 日替わり（UFO or バスケットゴール）。飛行中のボールが当たり判定円に入ると吸い込み演出（`targetActive`）
- **操作エリア**: 余白のある中央。待機ボールを上方向にドラッグ／フリックすると投擲。ドラッグ中は放物線の予測軌道を点線表示
- **下部プール**: 直近のエントリが物理で底に積まれ「溜まっていく」入口を表現
- **ピッカー**: 8感情を横スクロールで選択＋一言メモ入力

**投擲フロー**:
```
release(上方向) → consumeThrow() で枠確認
  → 枠あり: addEntry() で永続化 → usePhysics.launch() で視覚的に発射
  → 枠なし: 何もしない（残数0表示のまま、トーンを保つ）
```
※ 記録は **投擲時点で確定** する（飛行中にアプリが閉じても残る）。物理は演出。

### 6.2 アーカイブ画面（ArchiveScreen）
- 追加画面から **下にスクロール** して到達
- 画面いっぱいに密パックされたボール（`packBalls`）
- 新しい日付が上。日付の境界に薄い点線、右側に小さなピル型ラベル（例 `5/18`）
- 説明文なし・空白最小

## 7. 主要ロジック

### 7.1 物理（`usePhysics.ts`）
- `matter-js` の World に地面・左右壁を静的ボディとして配置
- `requestAnimationFrame` ループで `Engine.update` → 各ボディ位置をReact stateにスナップショット
- `launch()`: 発射点から速度ベクトルでボールを生成（飛行フラグ）
- ターゲット当たり判定: 飛行ボールが当たり判定円に入ると下方へ弾き、`targetActive` を立てる
- 着地判定: 下部に達し十分減速したら `landed` 化（記録は別途投擲時に確定済み）
- `seedPool()`: 既存エントリの直近分を下部に撒いて初期の溜まりを再現

### 7.2 投擲回数制限（`rateLimit.ts`）
- 仕様: 最大10投、3時間ごとに回復
- 実装: 直近3時間以内の投擲タイムスタンプ数が10未満なら投擲可（スライディングウィンドウ）。
  3時間より古い投擲は自動的に枠が回復したとみなす

### 7.3 アーカイブ密パック（`packBalls.ts`）
- 日付降順でグルーピング
- 行ごとに半セルずらすヘックス配置＋id由来の微小ジッターで「ギチギチ感」を演出
- 各日の帯の上端・中心Yを返し、点線とピルの配置に使用

### 7.4 日替わりインタラクション（`targets/index.ts`）
- 年内通算日（`dayOfYear`）をシードにターゲットを切替（UFO / バスケット）
- `TargetConfig` で位置（相対）と当たり判定半径を定義。`?ブロック`等は同インターフェースで追加可能

## 8. 素材パイプライン

支給されたスプライトシート3枚（POSITIVE / NEUTRAL / NEGATIVE）から、Pillow を用いて
個別ボールを抽出した。

1. アルファ（焼き込み市松背景）ではなく **彩度・輝度マスク** でボール領域を検出
2. 行バンド→列バンドの順でグリッドを特定（4列）
3. 各セルの重心・半径から **円形アルファマスク**（軽いフェザー）で切り抜き、256pxの透過PNGに統一
4. `assets/balls/<emotion>_<0..3>.png` として保存（計32枚）
5. 各感情の代表色は素材リングからサンプリングして `emotions.ts` に反映

> 円形マスク方式により、背景色とほぼ同色のクリーム系ボール（落ち着く/リラックス）も
> クリーンに切り抜ける。
